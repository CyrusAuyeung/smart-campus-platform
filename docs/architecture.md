# 系统架构说明

## 总览

本系统采用模块化单体 + 异步基础设施方案。

- 前端：Next.js，负责统一体验、响应式与无障碍
- 后端：Spring Boot，负责领域建模、事务一致性、规则引擎
- 数据层：PostgreSQL + Redis + RabbitMQ
- 运维层：Docker Compose + GitHub Actions + Prometheus/Grafana

## 业务模块

### 空间预约

- 学术空间：连续时间区间 + 5 分钟隐形缓冲
- 体育设施：离散槽位 + 场地组合拓扑

### 热门活动

- Redis 库存预热
- Lua 原子扣减
- MQ 削峰落单
- 补偿任务回滚库存

### 状态机

- 订单状态由统一应用服务迁移
- 延迟消息负责超时取消
- 支付回调和超时取消通过乐观锁竞态仲裁

### 规则引擎

- 责任链模式
- 数据库存储规则配置
- 前端展示命中原因，提升可解释性

## 数据建模原则

- 统一订单，不统一资源占用模型
- 能用数据库约束解决的问题，不只依赖应用层判断
- 所有关键状态迁移都要有日志与审计记录
